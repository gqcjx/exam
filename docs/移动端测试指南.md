# 移动端界面测试指南

## 方法一：使用浏览器开发者工具（推荐）

### Chrome/Edge 浏览器

1. **打开开发者工具**：
   - 按 `F12` 或 `Ctrl+Shift+I`（Windows/Linux）
   - 按 `Cmd+Option+I`（Mac）

2. **切换到设备模拟模式**：
   - 点击工具栏中的设备图标（📱）或按 `Ctrl+Shift+M`
   - 或点击右上角的三个点菜单 → More tools → Toggle device toolbar

3. **选择设备**：
   - 在顶部设备选择器中选择设备（如 iPhone 12 Pro、Samsung Galaxy S20 等）
   - 或自定义尺寸（如 375x812）

4. **刷新页面**：
   - 刷新页面后即可看到移动端界面

### Firefox 浏览器

1. 按 `F12` 打开开发者工具
2. 点击右上角的响应式设计模式图标（📱）
3. 选择设备或自定义尺寸

## 方法二：使用 URL 参数强制启用移动端（最简单）

在浏览器地址栏的 URL 后添加 `?mobile=true`：

```
http://localhost:5173/login?mobile=true
http://localhost:5173/dashboard?mobile=true
```

**优点**：
- 无需打开开发者工具
- 可以在任何浏览器上测试
- 不影响其他功能

## 方法三：使用 localStorage 强制启用（持久化）

在浏览器控制台（F12）中执行：

```javascript
// 启用移动端模式
localStorage.setItem('forceMobile', 'true')

// 刷新页面
location.reload()
```

**取消移动端模式**：
```javascript
localStorage.removeItem('forceMobile')
location.reload()
```

**优点**：
- 设置后会在所有页面生效
- 刷新页面后仍然有效
- 直到清除 localStorage 或手动关闭

## 方法四：在真实手机上测试（最准确）

### 1. 确保手机和电脑在同一网络

### 2. 获取电脑的 IP 地址

**Windows**：
```bash
ipconfig
```
查找 "IPv4 地址"，例如：`192.168.1.100`

**Mac/Linux**：
```bash
ifconfig
# 或
ip addr
```

### 3. 启动开发服务器

确保 Vite 开发服务器允许外部访问：

```bash
npm run dev -- --host
```

或修改 `vite.config.ts`：
```typescript
export default defineConfig({
  server: {
    host: '0.0.0.0', // 允许外部访问
    port: 5173
  }
})
```

### 4. 在手机上访问

在手机浏览器中输入：
```
http://192.168.1.100:5173/login
```
（将 IP 地址替换为你的实际 IP）

## 方法五：使用 ngrok 等内网穿透工具（远程测试）

如果需要远程访问（不在同一网络），可以使用 ngrok：

1. **安装 ngrok**：
   ```bash
   npm install -g ngrok
   ```

2. **启动开发服务器**：
   ```bash
   npm run dev
   ```

3. **启动 ngrok**：
   ```bash
   ngrok http 5173
   ```

4. **使用 ngrok 提供的 URL**：
   在手机浏览器中访问 ngrok 提供的 URL（如 `https://xxxx.ngrok.io`）

## 推荐测试流程

1. **开发阶段**：使用方法二（URL 参数）快速测试
2. **调试阶段**：使用方法一（开发者工具）详细调试
3. **最终测试**：使用方法四（真实手机）验证实际效果

## 常见问题

### Q: URL 参数方法不生效？
A: 确保在登录页面添加参数，因为设备检测在页面加载时执行。

### Q: localStorage 方法不生效？
A: 确保在页面加载前设置，或设置后刷新页面。

### Q: 开发者工具模拟不准确？
A: 尝试选择真实的设备预设，或使用真实手机测试。

### Q: 如何同时测试桌面端和移动端？
A: 打开两个浏览器窗口，一个正常访问，一个添加 `?mobile=true` 参数。
